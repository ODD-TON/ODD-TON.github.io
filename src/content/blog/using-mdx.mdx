这个项目已经预装了 MDX 集成，所以除了普通 Markdown 文章，后面也可以逐步加入一些更灵活的内容形态。

## 可能的用法

- 在文章中插入自定义组件。
- 展示更结构化的提示框或对比块。
- 给项目复盘页加入更清晰的模块化布局。

## 目前为什么没有立刻大量使用

现在的核心任务是先把站点信息补齐并顺利上线，所以内容层仍然以简单、稳定的 Markdown 为主。等站点更新频率稳定之后，再考虑把 MDX 用在更合适的地方。

## 一个原则

技术能力保留着就好，但不要让“以后也许会用到”变成当前迟迟不发布的理由。
