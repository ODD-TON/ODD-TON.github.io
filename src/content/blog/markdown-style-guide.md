这个博客不会追求“什么都写”，而是优先写几类最容易积累价值的内容。

## 1. 项目推进记录

记录一个页面、一个小工具、一次重构是怎么从想法变成可交付结果的。

## 2. 部署与维护笔记

包括 GitHub Pages、CI 配置、静态站生成、目录结构整理等内容。

## 3. 技术学习总结

不是抄文档，而是把实际用过之后还值得保留的经验重新整理出来。

## 4. 阶段性回顾

有些内容不一定是教程，但可以帮助后续回看“当时为什么这么做”。

当站点有了明确的内容边界之后，更新成本反而会变低，因为每次写东西都知道应该往哪里放。
